## 维护规则

- 每篇论文只设置一个主方向，用于统计“主方向论文数”。
- 如果一篇论文跨多个方向，可以在其他方向中放到“Related Papers”小节，但不重复计入主方向论文数。
- 方向 slug 使用小写英文和连字符，例如 `graph-neural-networks`。
- 论文表格按年份倒序排列，即最新论文放在最前面。
- `Paper` 链接优先使用出版社页面、会议官网页面、DOI 页面或 arXiv 页面。
- `Code` 链接统一指向实验室账号下的 fork 仓库。
- 暂无 paper 或 code 时，统一写 `TODO`，不要混用 `todo`、`无`、`N/A`。

# EN:
# Index of Published Papers and Code from the Lab

This repository is used to maintain an index of the papers and code from the lab's published papers.

All code links point by default to source code repositories forked or maintained under the lab account.

## Overview of Research Directions

| Research Direction | Slug | Number of Papers in Main Research Direction |

| [Multivariate Time Series Forecasting](./directions/multivariate-time-series-forecasting/README.md) | `multivariate-time-series-forecasting` | 3 |

## Maintenance Rules

- Each paper should have only one main research direction for counting "papers in the main research direction".

- If a paper spans multiple research directions, it can be placed in the "Related Papers" section within other research directions, but it will not be counted twice in the main research direction count.

- Use lowercase letters and hyphens for research direction slugs, e.g., `graph-neural-networks`.

- Paper tables should be arranged in reverse chronological order, with the most recent paper at the top.

- `Paper` links should prioritize publisher pages, conference websites, DOI pages, or arXiv pages.

- `Code` links should consistently point to the fork repository under the lab account.

- When there is no paper or code available, consistently use `TODO`. Do not mix `todo`, `none`, or `N/A`.
